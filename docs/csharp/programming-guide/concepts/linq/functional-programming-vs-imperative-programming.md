## <a name="advantages-of-pure-functions"></a><span data-ttu-id="b4405-149">Vantaggi delle funzioni pure</span><span class="sxs-lookup"><span data-stu-id="b4405-149">Advantages of Pure Functions</span></span>  
 <span data-ttu-id="b4405-150">Il motivo principale per implementare le trasformazioni funzionali come funzioni pure è che le funzioni pure sono componibili, ossia sono autonome e senza stato.</span><span class="sxs-lookup"><span data-stu-id="b4405-150">The primary reason to implement functional transformations as pure functions is that pure functions are composable: that is, self-contained and stateless.</span></span> <span data-ttu-id="b4405-151">Queste caratteristiche offrono numerosi vantaggi, tra cui quelli riportati di seguito:</span><span class="sxs-lookup"><span data-stu-id="b4405-151">These characteristics bring a number of benefits, including the following:</span></span>  
  
- <span data-ttu-id="b4405-152">Maggiore facilità di lettura e manutenibilità,</span><span class="sxs-lookup"><span data-stu-id="b4405-152">Increased readability and maintainability.</span></span> <span data-ttu-id="b4405-153">in quanto ogni funzione è progettata per realizzare un'attività specifica con gli argomenti assegnati.</span><span class="sxs-lookup"><span data-stu-id="b4405-153">This is because each function is designed to accomplish a specific task given its arguments.</span></span> <span data-ttu-id="b4405-154">La funzione non si basa su alcuno stato esterno.</span><span class="sxs-lookup"><span data-stu-id="b4405-154">The function does not rely on any external state.</span></span>  
  
- <span data-ttu-id="b4405-155">Sviluppo iterativo più semplice,</span><span class="sxs-lookup"><span data-stu-id="b4405-155">Easier reiterative development.</span></span> <span data-ttu-id="b4405-156">in quanto è più agevole eseguire il refactoring del codice ed è più facile implementare le modifiche di progettazione.</span><span class="sxs-lookup"><span data-stu-id="b4405-156">Because the code is easier to refactor, changes to design are often easier to implement.</span></span> <span data-ttu-id="b4405-157">Si supponga ad esempio di scrivere una trasformazione complicata e poi di realizzare che una parte di codice viene ripetuta diverse volte al suo interno.</span><span class="sxs-lookup"><span data-stu-id="b4405-157">For example, suppose you write a complicated transformation, and then realize that some code is repeated several times in the transformation.</span></span> <span data-ttu-id="b4405-158">Se si esegue il refactoring tramite un metodo puro, è possibile chiamare il metodo puro quando è necessario senza preoccuparsi degli effetti collaterali.</span><span class="sxs-lookup"><span data-stu-id="b4405-158">If you refactor through a pure method, you can call your pure method at will without worrying about side effects.</span></span>

 <span data-ttu-id="b4405-162">Nella tradizionale programmazione orientata a oggetti (OOP, Object-Oriented Programming), gli sviluppatori sono in genere abituati a programmare nello stile imperativo/procedurale.</span><span class="sxs-lookup"><span data-stu-id="b4405-162">In traditional object-oriented programming (OOP), most developers are accustomed to programming in the imperative/procedural style.</span></span> <span data-ttu-id="b4405-163">Per passare allo sviluppo in uno stile funzionale puro, devono modificare il loro modo di ragionare e l'approccio adottato per lo sviluppo.</span><span class="sxs-lookup"><span data-stu-id="b4405-163">To switch to developing in a pure functional style, they have to make a transition in their thinking and their approach to development.</span></span>
