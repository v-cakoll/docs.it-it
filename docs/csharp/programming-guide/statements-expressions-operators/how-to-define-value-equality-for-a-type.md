# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="9eae2-102">Procedura: definire l'uguaglianza di valori per un tipo (Guida per programmatori C#)</span><span class="sxs-lookup"><span data-stu-id="9eae2-102">How to: Define Value Equality for a Type (C# Programming Guide)</span></span>
<span data-ttu-id="9eae2-103">Quando si definisce una classe o uno struct, si decide se è opportuno creare una definizione personalizzata di uguaglianza di valore, o equivalenza, per il tipo.</span><span class="sxs-lookup"><span data-stu-id="9eae2-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="9eae2-104">In genere, l'uguaglianza di valori viene implementata quando si prevede che oggetti del tipo vengano aggiunti a una raccolta, o quando lo scopo principale di tali oggetti consiste nell'archiviare un set di campi o di proprietà.</span><span class="sxs-lookup"><span data-stu-id="9eae2-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="9eae2-105">È possibile basare la definizione di uguaglianza di valori su un confronto di tutti i campi e di tutte le proprietà nel tipo, oppure su un sottoinsieme.</span><span class="sxs-lookup"><span data-stu-id="9eae2-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span> <span data-ttu-id="9eae2-106">Ma in entrambi i casi e in entrambe le classi e gli struct, l'implementazione deve seguire le cinque garanzie di equivalenza:</span><span class="sxs-lookup"><span data-stu-id="9eae2-106">But in either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence:</span></span>  
  
1.  <span data-ttu-id="9eae2-107">x.`Equals`(x) restituisce `true.`. Questa viene denominata proprietà riflessiva.</span><span class="sxs-lookup"><span data-stu-id="9eae2-107">x.`Equals`(x) returns `true.` This is called the reflexive property.</span></span>  
  
2.  <span data-ttu-id="9eae2-108">x.`Equals`(y) restituisce lo stesso valore di y.`Equals`(x).</span><span class="sxs-lookup"><span data-stu-id="9eae2-108">x.`Equals`(y) returns the same value as y.`Equals`(x).</span></span> <span data-ttu-id="9eae2-109">Questa viene denominata proprietà simmetrica.</span><span class="sxs-lookup"><span data-stu-id="9eae2-109">This is called the symmetric property.</span></span>  
  
3.  <span data-ttu-id="9eae2-110">Se (x.`Equals`(y) && y`Equals`(z)) restituisce `true`, x.`Equals`(z) restituisce `true`.</span><span class="sxs-lookup"><span data-stu-id="9eae2-110">if (x.`Equals`(y) && y.`Equals`(z)) returns `true`, then x.`Equals`(z) returns `true`.</span></span> <span data-ttu-id="9eae2-111">Questa viene denominata proprietà transitiva.</span><span class="sxs-lookup"><span data-stu-id="9eae2-111">This is called the transitive property.</span></span>  
  
4.  <span data-ttu-id="9eae2-112">Le successive chiamate di x.`Equals`(y) restituiscono lo stesso valore purché gli oggetti a cui x e y fanno riferimento non vengano modificati.</span><span class="sxs-lookup"><span data-stu-id="9eae2-112">Successive invocations of x.`Equals`(y) return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5.  <span data-ttu-id="9eae2-113">x.`Equals`(null) restituisce `false`.</span><span class="sxs-lookup"><span data-stu-id="9eae2-113">x.`Equals`(null) returns `false`.</span></span> <span data-ttu-id="9eae2-114">Tuttavia, null.Equals(null) genera un'eccezione in quanto non rispetta la regola numero 2 precedente.</span><span class="sxs-lookup"><span data-stu-id="9eae2-114">However, null.Equals(null) throws an exception; it does not obey rule number two above.</span></span>  
  
 <span data-ttu-id="9eae2-115">Gli struct definiti hanno già un'implementazione predefinita di uguaglianza di valore che eredita dall'override <xref:System.ValueType?displayProperty=fullName> del metodo <xref:System.Object.Equals%28System.Object%29?displayProperty=fullName>.</span><span class="sxs-lookup"><span data-stu-id="9eae2-115">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=fullName> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=fullName> method.</span></span> <span data-ttu-id="9eae2-116">Questa implementazione usa il processo di reflection per esaminare tutti i campi e tutte le proprietà nel tipo.</span><span class="sxs-lookup"><span data-stu-id="9eae2-116">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="9eae2-117">Sebbene questa implementazione produca risultati corretti, è relativamente lenta rispetto a un'implementazione personalizzata che viene scritta specificamente per il tipo.</span><span class="sxs-lookup"><span data-stu-id="9eae2-117">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="9eae2-118">I dettagli di implementazione per l'uguaglianza di valori sono diversi per le classi e gli struct.</span><span class="sxs-lookup"><span data-stu-id="9eae2-118">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="9eae2-119">Tuttavia, sia le classi che gli struct richiedono gli stessi passaggi di base per l'implementazione dell'uguaglianza:</span><span class="sxs-lookup"><span data-stu-id="9eae2-119">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1.  <span data-ttu-id="9eae2-120">Eseguire l'override del metodo di tipo [virtual](../../../csharp/language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=fullName>.</span><span class="sxs-lookup"><span data-stu-id="9eae2-120">Override the [virtual](../../../csharp/language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=fullName> method.</span></span> <span data-ttu-id="9eae2-121">Nella maggior parte dei casi l'implementazione di `bool Equals( object obj )` deve solo chiamare il metodo `Equals` specifico per il tipo, che è l'implementazione dell'interfaccia <xref:System.IEquatable%601?displayProperty=fullName>.</span><span class="sxs-lookup"><span data-stu-id="9eae2-121">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=fullName> interface.</span></span> <span data-ttu-id="9eae2-122">(Vedere il passaggio 2.)</span><span class="sxs-lookup"><span data-stu-id="9eae2-122">(See step 2.)</span></span>  
  
2.  <span data-ttu-id="9eae2-123">Implementare l'interfaccia <xref:System.IEquatable%601?displayProperty=fullName> definendo un metodo `Equals` specifico per il tipo.</span><span class="sxs-lookup"><span data-stu-id="9eae2-123">Implement the <xref:System.IEquatable%601?displayProperty=fullName> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="9eae2-124">È in questo passaggio che viene eseguito il confronto di equivalenza effettivo.</span><span class="sxs-lookup"><span data-stu-id="9eae2-124">This is where the actual class="sxs-lookup"><span data-stu-id="9eae2-125">For example, you might decide to define equality by comparing only one or two fields in your type.</span></span> <span data-ttu-id="9eae2-126">Non generare eccezioni da `Equals`.</span><span class="sxs-lookup"><span data-stu-id="9eae2-126">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="9eae2-127">Solo per le classi: questo metodo deve esaminare solo i campi che vengono dichiarati nella classe.</span><span class="sxs-lookup"><span data-stu-id="9eae2-127">For classes only: This method should examine only fields 3.  <span data-ttu-id="9eae2-130">Facoltativo ma consigliato: eseguire l'overload degli operatori [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) e [!=](../../../csharp/language-reference/operators/not-equal-operator.md).</span><span class="sxs-lookup"><span data-stu-id="9eae2-130">Optional but recommended: Overload the [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) and [!=](../../../csharp/language-reference/operators/not-equal-operator.md) operators.</span></span>  
  
4.  <span data-ttu-id="9eae2-131">Eseguire l'override di <xref:System.Object.GetHashCode%2A?displayProperty=fullName> in modo che due oggetti con uguaglianza di valori producano lo stesso codice hash.</span><span class="sxs-lookup"><span data-stu-id="9eae2-131">Override <xref:System.Object.GetHashCode%2A?displayProperty=fullName> so that two objects that have value equality produce the same hash code.</span></span>
