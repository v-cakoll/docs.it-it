# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="6b3c6-102">Avviso del compilatore (livello 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="6b3c6-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="6b3c6-103">Il campo privato 'field' è assegnato, ma il suo valore non viene mai usato</span><span class="sxs-lookup"><span data-stu-id="6b3c6-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="6b3c6-104">Questo avviso viene visualizzato in vai scenari in cui il compilatore può verificare che una variabile non viene mai usata come riferimento:</span><span class="sxs-lookup"><span data-stu-id="6b3c6-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
-   <span data-ttu-id="6b3c6-105">A un campo privato è assegnato un valore costante, che però non viene mai letto.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="6b3c6-106">L'assegnazione non necessaria può influire sulle prestazioni.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="6b3c6-107">Considerare la possibilità di rimuovere il campo.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-107">Consider removing the field.</span></span>  
  
-   <span data-ttu-id="6b3c6-108">A un campo statico privato o interno è assegnato un valore costante solo nell'inizializzatore.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="6b3c6-109">Considerare la possibilità di impostare il campo su un valore costante.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-109">Consider changing the field to a const.</span></span>  
  
-   <span data-ttu-id="6b3c6-110">A un campo privato o interno sono assegnati valori costanti e il campo viene usato solo in blocchi esclusi dalle direttive #ifdef.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="6b3c6-111">Considerare la possibilità di inserire il campo nel blocco #ifdef.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-111">Consider putting the field inside the #ifdef block.</span></span>  
  
-   <span data-ttu-id="6b3c6-112">A un campo privato o interno sono assegnati valori costanti in più percorsi, ma non si accede al campo in altri modi.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="6b3c6-113">Se il campo non è necessario, considerare la possibilità di rimuoverlo.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="6b3c6-114">In caso contrario, usarlo in modo appropriato.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="6b3c6-115">In altre situazioni o se la soluzione alternativa suggerita non è accettabile, usare #pragma 0414.</span><span class="sxs-lookup"><span data-stu-id="6b3c6-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="6b3c6-116">L'esempio seguente mostra un modo in cui viene generato l'errore CS0414:</span><span class="sxs-lookup"><span data-stu-id="6b3c6-116">The following sample shows one way in which CS0414 will be generated:</span></span>

```csharp  
{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="6b3c6-117">**Nota** Se la variabile `i` è dichiarata come `protected or public`non verrà generato un errore, in quanto il compilatore non sa se potrebbe essere usata da una classe derivata oppure se altro codice cliente potrebbe creare un'istanza della classe e fare riferimento alla variabile</span><span class="sxs-lookup"><span data-stu-id="6b3c6-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
