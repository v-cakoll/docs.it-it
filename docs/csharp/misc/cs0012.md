# <a name="compiler-error-cs0012"></a><span data-ttu-id="75376-102">Errore del compilatore CS0012</span><span class="sxs-lookup"><span data-stu-id="75376-102">Compiler Error CS0012</span></span>
<span data-ttu-id="75376-103">Il tipo 'type' è definito in un assembly di cui manca il riferimento.</span><span class="sxs-lookup"><span data-stu-id="75376-103">The type 'type' is defined in an assembly that is not referenced.</span></span> <span data-ttu-id="75376-104">Aggiungere un riferimento all'assembly 'assembly'.</span><span class="sxs-lookup"><span data-stu-id="75376-104">You must add a reference to assembly 'assembly'.</span></span>  
  
 <span data-ttu-id="75376-105">Non è stata trovata la definizione relativa a un tipo a cui si fa riferimento.</span><span class="sxs-lookup"><span data-stu-id="75376-105">The definition for a referenced type was not found.</span></span> <span data-ttu-id="75376-106">Questo errore può verificarsi se nella compilazione non è inclusa una DLL obbligatoria.</span><span class="sxs-lookup"><span data-stu-id="75376-106">This could occur if a required .DLL file is not included in the compilation.</span></span> <span data-ttu-id="75376-107">Per altre informazioni, vedere [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) e [-reference (C# opzioni del compilatore)](../../csharp/language-reference/compiler-options/reference-compiler-option.md).</span><span class="sxs-lookup"><span data-stu-id="75376-107">For more information, see [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) and [/reference (C# Compiler Options)](../../csharp/language-reference/compiler-options/reference-compiler-option.md).</span></span>  
  
 <span data-ttu-id="75376-108">La sequenza di compilazioni genera seguente l'errore CS0012:</span><span class="sxs-lookup"><span data-stu-id="75376-108">The following sequence of compilations will result in CS0012:</span></span>  
  
```csharp  
// cs0012a.cs  
// compile with: /target:library  
public class A {}  
```  
  
 <span data-ttu-id="75376-109">Quindi:</span><span class="sxs-lookup"><span data-stu-id="75376-109">Then:</span></span>  
  
```csharp  
// cs0012b.cs  
// compile with: /target:library /reference:cs0012a.dll  
public class B  
{  
   public static A f()  
   {  
      return new A();  
   }  
}  
```  
  
 <span data-ttu-id="75376-110">Quindi:</span><span class="sxs-lookup"><span data-stu-id="75376-110">Then:</span></span>  
  
```csharp  
// cs0012c.cs  
// compile with: /reference:cs0012b.dll  
class C  
{  
   public static void Main()  
   {  
      object o = B.f();   // CS0012  
   }  
}  
```  
  
 <span data-ttu-id="75376-111">L'errore CS0012 può essere risolto eseguendo la compilazione con `/reference:cs0012b.dll;cs0012a.dll`, oppure mediante la [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) in Visual Studio, per aggiungere un riferimento a cs0012a.dll oltre a cs0012b.dll.</span><span class="sxs-lookup"><span data-stu-id="75376-111">You could resolve this CS0012 by compiling with `/reference:cs0012b.dll;cs0012a.dll`, or in Visual Studio by using the [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) to add a reference to cs0012a.dll in addition to cs0012b.dll.</span></span>
