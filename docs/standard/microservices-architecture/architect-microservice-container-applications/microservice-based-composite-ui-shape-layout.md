# <a name="creating-composite-ui-based-on-microservices-including-visual-ui-shape-and-layout-generated-by-multiple-microservices"></a><span data-ttu-id="69ada-103">Creazione dell'interfaccia utente composita basata su microservizi, tra cui la forma visiva dell'interfaccia utente e il layout generato da più microservizi</span><span class="sxs-lookup"><span data-stu-id="69ada-103">Creating composite UI based on microservices, including visual UI shape and layout generated by multiple microservices</span></span>

<span data-ttu-id="69ada-104">L'architettura dei microservizi inizia spesso con la gestione dei dati e la logica lato server.</span><span class="sxs-lookup"><span data-stu-id="69ada-104">Microservices architecture often starts with the server side handling data and logic.</span></span> <span data-ttu-id="69ada-105">Un approccio più avanzato prevede tuttavia la progettazione dell'interfaccia utente dell'applicazione basata anche sui microservizi.</span><span class="sxs-lookup"><span data-stu-id="69ada-105">However, a more advanced approach is to design your application UI based on microservices as well.</span></span> <span data-ttu-id="69ada-106">Ciò significa avere un'interfaccia utente composita generata dai microservizi, invece di avere i microservizi sul server e solo un'app client monolitica che utilizza i microservizi.</span><span class="sxs-lookup"><span data-stu-id="69ada-106">That means having a composite UI produced by the microservices, instead of having microservices on the server and just a monolithic client app consuming the microservices.</span></span> <span data-ttu-id="69ada-107">Con questo approccio, i microservizi creati possono essere completati con la rappresentazione sia logica che visiva.</span><span class="sxs-lookup"><span data-stu-id="69ada-107">With this approach, the microservices you build can be complete with both logic and visual representation.</span></span>

<span data-ttu-id="69ada-108">La figura 4-20 illustra l'approccio più semplice in cui i microservizi vengono utilizzati da un'applicazione client monolitica.</span><span class="sxs-lookup"><span data-stu-id="69ada-108">Figure 4-20 shows the simpler approach of just consuming microservices from a monolithic client application.</span></span> <span data-ttu-id="69ada-109">Nel mezzo è ovviamente possibile avere un servizio MVC ASP.NET che genera il codice HTML e JavaScript.</span><span class="sxs-lookup"><span data-stu-id="69ada-109">Of course, you could have an ASP.NET MVC service in between producing the HTML and JavaScript.</span></span> <span data-ttu-id="69ada-110">La figura è una rappresentazione semplificata che evidenzia la presenza di una sola interfaccia utente client (monolitica) che utilizza i microservizi, concentrati solo sulla logica e sui dati e non sulla forma dell'interfaccia utente (HTML e JavaScript).</span><span class="sxs-lookup"><span data-stu-id="69ada-110">The figure is a simplification that highlights that you have a single (monolithic) client UI consuming the microservices, which just focus on logic and data and not on the UI shape (HTML and JavaScript).</span></span>

<span data-ttu-id="69ada-111">**Figura 4-20**.</span><span class="sxs-lookup"><span data-stu-id="69ada-111">**Figure 4-20**.</span></span> <span data-ttu-id="69ada-112">Applicazione dell'interfaccia utente monolitica che utilizza i microservizi back-end</span><span class="sxs-lookup"><span data-stu-id="69ada-112">A monolithic UI application consuming back-end microservices</span></span>

<span data-ttu-id="69ada-113">Un'interfaccia utente composita viene invece generata con precisione ed è composta dai microservizi stessi.</span><span class="sxs-lookup"><span data-stu-id="69ada-113">In contrast, a composite UI is precisely generated and composed by the microservices themselves.</span></span> <span data-ttu-id="69ada-114">Alcuni dei microservizi gestiscono la forma visiva di aree specifiche dell'interfaccia utente.</span><span class="sxs-lookup"><span data-stu-id="69ada-114">Some of the microservices drive the visual shape of specific areas of the UI.</span></span> <span data-ttu-id="69ada-115">La differenza principale è che si hanno componenti dell'interfaccia utente client (ad esempio, le classi TS) basati su modelli e che l'elemento ViewModel dell'interfaccia utente di data shaping di tali modelli deriva da ogni microservizio.</span><span class="sxs-lookup"><span data-stu-id="69ada-115">The key difference is that you have client UI components (TS classes, for example) based on templates, and the data-shaping-UI ViewModel for those templates comes from each microservice.</span></span>

<span data-ttu-id="69ada-116">Durante l'avvio dell'applicazione client, ognuno dei componenti dell'interfaccia utente client (ad esempio, le classi TypeScript) si registra con un microservizio dell'infrastruttura in grado di fornire ViewModel per un determinato scenario.</span><span class="sxs-lookup"><span data-stu-id="69ada-116">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="69ada-117">Se la forma del microservizio cambia, anche l'interfaccia utente cambia.</span><span class="sxs-lookup"><span data-stu-id="69ada-117">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="69ada-118">La figura 4-21 illustra una versione di questo approccio basato sull'interfaccia utente composita.</span><span class="sxs-lookup"><span data-stu-id="69ada-118">Figure 4-21 shows a version of this composite UI approach.</span></span> <span data-ttu-id="69ada-119">La figura è semplificata, perché è possibile avere altri microservizi che aggregano parti granulari in base a tecniche diverse, a seconda del fatto che si stia creando un tradizionale approccio Web (MVC ASP.NET) o un'applicazione a pagina singola.</span><span class="sxs-lookup"><span data-stu-id="69ada-119">This is simplified, because you might have other microservices that are aggregating granular parts based on different techniques—it depends on whether you are building a traditional web approach (ASP.NET MVC) or an SPA (Single Page Application).</span></span>

<span data-ttu-id="69ada-124">Un approccio basato sull'interfaccia utente composita gestito dai microservizi può essere più o meno complesso a seconda delle tecnologie usate per l'interfaccia utente.</span><span class="sxs-lookup"><span data-stu-id="69ada-124">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="69ada-125">Per compilare una tradizionale applicazione Web, ad esempio, non si useranno le stesse tecniche usate per compilare un'applicazione a pagina singola o per le app per dispositivi mobili native (come nello sviluppo di app Xamarin, che può essere più complesso per questo approccio).</span><span class="sxs-lookup"><span data-stu-id="69ada-125">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>

<span data-ttu-id="69ada-126">L'applicazione di esempio [eShopOnContainers](https://aka.ms/MicroservicesArchitecture) usa l'approccio basato sull'interfaccia utente monolitica per più motivi.</span><span class="sxs-lookup"><span data-stu-id="69ada-126">The [eShopOnContainers](https://aka.ms/MicroservicesArchitecture) sample application uses the monolithic UI approach for multiple reasons.</span></span> <span data-ttu-id="69ada-127">Prima di tutto, è un'introduzione a microservizi e contenitori.</span><span class="sxs-lookup"><span data-stu-id="69ada-127">First, it is an introduction to microservices and containers.</span></span> <span data-ttu-id="69ada-128">Un'interfaccia utente composita è più avanzata, ma presenta anche un livello di complessità più elevato quando si progetta e sviluppa l'interfaccia utente.</span><span class="sxs-lookup"><span data-stu-id="69ada-128">A composite UI is more advanced but also requires further complexity when designing and developing the UI.</span></span> <span data-ttu-id="69ada-129">In secondo luogo, eShopOnContainers fornisce anche un'app per dispositivi mobili nativa basata su Xamarin, che ne aumenterà la complessità sul lato C\# del client.</span><span class="sxs-lookup"><span data-stu-id="69ada-129">Second, eShopOnContainers also provides a native mobile app based on Xamarin, which would make it more complex on the client C\# side.</span></span>
