## <a name="drawbacks-of-the-api-gateway-pattern"></a><span data-ttu-id="0cb0f-248">Svantaggi dello schema API Gateway</span><span class="sxs-lookup"><span data-stu-id="0cb0f-248">Drawbacks of the API Gateway pattern</span></span>

- <span data-ttu-id="0cb0f-249">Lo svantaggio più importante consiste nel fatto che, quando si implementa un gateway API, tale livello viene accoppiato con i microservizi interni.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-249">The most important drawback is that when you implement an API Gateway, you're coupling that tier with the internal microservices.</span></span> <span data-ttu-id="0cb0f-250">Un accoppiamento di questo tipo potrebbe causare gravi difficoltà per l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-250">Coupling like this might introduce serious difficulties for your application.</span></span> <span data-ttu-id="0cb0f-251">Clemens Vaster, architetto del team del bus di servizio di Azure, chiama questa difficoltà potenziale "il nuovo ESB" nella sessione sulla [messaggistica e i microservizi](https://www.youtube.com/watch?v=rXi5CLjIQ9k) di GOTO 2016.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-251">Clemens Vaster, architect at the Azure Service Bus team, refers to this potential difficulty as "the new ESB" in the "[Messaging and Microservices](https://www.youtube.com/watch?v=rXi5CLjIQ9k)" session at GOTO 2016.</span></span>

- <span data-ttu-id="0cb0f-256">Un gateway API richiede un costo di sviluppo aggiuntivo e ulteriori operazioni di manutenzione future se include la logica personalizzata e l'aggregazione dei dati.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-256">An API Gateway requires additional development cost and future maintenance if it includes custom logic and data aggregation.</span></span> <span data-ttu-id="0cb0f-257">Gli sviluppatori devono aggiornare il gateway API per esporre gli endpoint di ogni microservizio.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-257">Developers must update the API Gateway in order to expose each microservice's endpoints.</span></span> <span data-ttu-id="0cb0f-258">Inoltre, le modifiche di implementazione nei microservizi interni potrebbero causare modifiche al codice al livello del gateway API.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-258">Moreover, implementation changes in the internal microservices might cause code changes at the API Gateway level.</span></span> <span data-ttu-id="0cb0f-259">Tuttavia, se il gateway API viene usato semplicemente per le attività di sicurezza, registrazione e controllo delle versioni, come avviene con Gestione API di Azure, il costo di sviluppo aggiuntivo potrebbe non essere applicato.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-259">However, if the API Gateway is just applying security, logging, and versioning (as when using Azure API Management), this additional development cost might not apply.</span></span>

- <span data-ttu-id="0cb0f-260">Se il gateway API viene sviluppato da un singolo team, potrebbe verificarsi un collo di bottiglia nello sviluppo.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-260">If the API Gateway is developed by a single team, there can be a development bottleneck.</span></span> <span data-ttu-id="0cb0f-261">Ecco un altro motivo per cui è preferibile avere più gateway API specifici che rispondono alle diverse esigenze dei client.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-261">This is another reason why a better approach is to have several fined-grained API Gateways that respond to different client needs.</span></span> <span data-ttu-id="0cb0f-262">È anche possibile suddividere internamente il gateway API in più aree o livelli di proprietà dei diversi team che lavorano sui microservizi interni.</span><span class="sxs-lookup"><span data-stu-id="0cb0f-262">You could also segregate the API Gateway internally into multiple areas or layers that are owned by the different teams working on the internal microservices.</span></span>
