# <a name="property-design"></a><span data-ttu-id="4f58e-102">Progettazione di proprietà</span><span class="sxs-lookup"><span data-stu-id="4f58e-102">Property Design</span></span>
<span data-ttu-id="4f58e-103">Sebbene le proprietà siano tecnicamente molto simili ai metodi, sono piuttosto diverse in termini di scenari di utilizzo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="4f58e-104">Dovrebbero essere considerati campi intelligenti.</span><span class="sxs-lookup"><span data-stu-id="4f58e-104">They should be seen as smart fields.</span></span> <span data-ttu-id="4f58e-105">Hanno la sintassi di chiamata dei campi e la flessibilità dei metodi.</span><span class="sxs-lookup"><span data-stu-id="4f58e-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="4f58e-106">✔️ creare proprietà solo Get se il chiamante non deve essere in grado di modificare il valore della proprietà.</span><span class="sxs-lookup"><span data-stu-id="4f58e-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="4f58e-107">Tenere presente che se il tipo della proprietà è un tipo di riferimento modificabile, è possibile modificare il valore della proprietà anche se la proprietà è solo Get.</span><span class="sxs-lookup"><span data-stu-id="4f58e-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="4f58e-108">❌ non forniscono proprietà o proprietà solo set con il metodo set con accessibilità più ampia rispetto al Getter.</span><span class="sxs-lookup"><span data-stu-id="4f58e-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="4f58e-109">Ad esempio, non usare le proprietà con un setter pubblico e un getter protetto.</span><span class="sxs-lookup"><span data-stu-id="4f58e-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="4f58e-110">Se non è possibile specificare il getter della proprietà, implementare invece la funzionalità come metodo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="4f58e-111">Prendere in considerazione l'avvio del nome del metodo con `Set` e seguire il nome della proprietà.</span><span class="sxs-lookup"><span data-stu-id="4f58e-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="4f58e-112">Ad esempio, <xref:System.AppDomain> dispone di un metodo denominato `SetCachePath` invece di avere una proprietà solo set denominata `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="4f58e-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="4f58e-113">✔️ forniscono valori predefiniti ragionevoli per tutte le proprietà, assicurando che le impostazioni predefinite non comportino un problema di sicurezza o un codice inefficiente.</span><span class="sxs-lookup"><span data-stu-id="4f58e-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="4f58e-114">✔️ consentono di impostare le proprietà in qualsiasi ordine anche se ciò comporta uno stato temporaneo non valido dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="4f58e-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="4f58e-115">È comune che due o più proprietà siano correlate a un punto in cui alcuni valori di una proprietà potrebbero non essere validi in base ai valori di altre proprietà nello stesso oggetto.</span><span class="sxs-lookup"><span data-stu-id="4f58e-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="4f58e-116">In questi casi, le eccezioni derivanti dallo stato non valido devono essere posticipate fino a quando le proprietà correlate non vengono effettivamente utilizzate insieme dall'oggetto.</span><span class="sxs-lookup"><span data-stu-id="4f58e-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="4f58e-117">✔️ conservare il valore precedente se un metodo di impostazione della proprietà genera un'eccezione.</span><span class="sxs-lookup"><span data-stu-id="4f58e-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="4f58e-118">❌ evitare di generare eccezioni dai getter della proprietà.</span><span class="sxs-lookup"><span data-stu-id="4f58e-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="4f58e-119">I getter della proprietà devono essere semplici operazioni e non devono avere precondizioni.</span><span class="sxs-lookup"><span data-stu-id="4f58e-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="4f58e-120">Se un getter può generare un'eccezione, dovrebbe essere probabilmente riprogettato per essere un metodo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="4f58e-121">Si noti che questa regola non si applica agli indicizzatori, in cui si prevedono eccezioni come risultato della convalida degli argomenti.</span><span class="sxs-lookup"><span data-stu-id="4f58e-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="4f58e-122">Progettazione proprietà indicizzata</span><span class="sxs-lookup"><span data-stu-id="4f58e-122">Indexed Property Design</span></span>
 <span data-ttu-id="4f58e-123">Una proprietà indicizzata è una proprietà speciale che può avere parametri e può essere chiamata con una sintassi speciale simile all'indicizzazione della matrice.</span><span class="sxs-lookup"><span data-stu-id="4f58e-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="4f58e-124">Le proprietà indicizzate sono comunemente definite indicizzatori.</span><span class="sxs-lookup"><span data-stu-id="4f58e-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="4f58e-125">Gli indicizzatori devono essere usati solo nelle API che consentono di accedere agli elementi di una raccolta logica.</span><span class="sxs-lookup"><span data-stu-id="4f58e-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="4f58e-126">Ad esempio, una stringa è una raccolta di caratteri e l'indicizzatore in <xref:System.String?displayProperty=nameWithType> è stato aggiunto per accedere ai relativi caratteri.</span><span class="sxs-lookup"><span data-stu-id="4f58e-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="4f58e-127">✔️ CONSIGLIABILE usare gli indicizzatori per fornire l'accesso ai dati archiviati in una matrice interna.</span><span class="sxs-lookup"><span data-stu-id="4f58e-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="4f58e-128">✔️ considerare la possibilità di fornire indicizzatori sui tipi che rappresentano raccolte di elementi.</span><span class="sxs-lookup"><span data-stu-id="4f58e-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="4f58e-129">❌ evitare di utilizzare proprietà indicizzate con più di un parametro.</span><span class="sxs-lookup"><span data-stu-id="4f58e-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="4f58e-130">Se la progettazione richiede più parametri, riconsiderare se la proprietà rappresenta effettivamente una funzione di accesso a una raccolta logica.</span><span class="sxs-lookup"><span data-stu-id="4f58e-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4f58e-131">In caso contrario, utilizzare i metodi.</span><span class="sxs-lookup"><span data-stu-id="4f58e-131">If it does not, use methods instead.</span></span> <span data-ttu-id="4f58e-132">Provare ad avviare il nome del metodo con `Get` o `Set`.</span><span class="sxs-lookup"><span data-stu-id="4f58e-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="4f58e-133">❌ evitare gli indicizzatori con tipi di parametro diversi da <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>o un'enumerazione.</span><span class="sxs-lookup"><span data-stu-id="4f58e-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="4f58e-134">Se la progettazione richiede altri tipi di parametri, valutare fortemente se l'API rappresenta effettivamente una funzione di accesso a una raccolta logica.</span><span class="sxs-lookup"><span data-stu-id="4f58e-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4f58e-135">In caso contrario, usare un metodo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-135">If it does not, use a method.</span></span> <span data-ttu-id="4f58e-136">Provare ad avviare il nome del metodo con `Get` o `Set`.</span><span class="sxs-lookup"><span data-stu-id="4f58e-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="4f58e-137">✔️ utilizzare il nome `Item` per le proprietà indicizzate a meno che non esista un nome ovviamente migliore (ad esempio, vedere la proprietà <xref:System.String.Chars%2A> in `System.String`).</span><span class="sxs-lookup"><span data-stu-id="4f58e-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="4f58e-138">In C#gli indicizzatori sono elementi denominati per impostazione predefinita.</span><span class="sxs-lookup"><span data-stu-id="4f58e-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="4f58e-139">Il <xref:System.Runtime.CompilerServices.IndexerNameAttribute> può essere utilizzato per personalizzare questo nome.</span><span class="sxs-lookup"><span data-stu-id="4f58e-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="4f58e-140">❌ non forniscono un indicizzatore e metodi semanticamente equivalenti.</span><span class="sxs-lookup"><span data-stu-id="4f58e-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="4f58e-141">❌ non forniscono più di una famiglia di indicizzatori in overload in un unico tipo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="4f58e-142">Questa operazione viene applicata dal C# compilatore.</span><span class="sxs-lookup"><span data-stu-id="4f58e-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="4f58e-143">❌ non utilizzano proprietà indicizzate non predefinite.</span><span class="sxs-lookup"><span data-stu-id="4f58e-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="4f58e-144">Questa operazione viene applicata dal C# compilatore.</span><span class="sxs-lookup"><span data-stu-id="4f58e-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="4f58e-145">Eventi di notifica della modifica delle proprietà</span><span class="sxs-lookup"><span data-stu-id="4f58e-145">Property Change Notification Events</span></span>
 <span data-ttu-id="4f58e-146">A volte è utile fornire un evento per notificare all'utente le modifiche apportate al valore di una proprietà.</span><span class="sxs-lookup"><span data-stu-id="4f58e-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="4f58e-147">Ad esempio, `System.Windows.Forms.Control` genera un evento `TextChanged` dopo che il valore della relativa proprietà `Text` è stato modificato.</span><span class="sxs-lookup"><span data-stu-id="4f58e-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="4f58e-148">✔️ valutare la possibilità di generare eventi di notifica delle modifiche quando vengono modificati i valori delle proprietà nelle API di alto livello, in genere componenti di progettazione.</span><span class="sxs-lookup"><span data-stu-id="4f58e-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="4f58e-149">Se è disponibile uno scenario valido che consente a un utente di conoscere quando una proprietà di un oggetto viene modificata, l'oggetto deve generare un evento di notifica delle modifiche per la proprietà.</span><span class="sxs-lookup"><span data-stu-id="4f58e-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="4f58e-150">Tuttavia, è improbabile che valga l'overhead per generare eventi di questo tipo per le API di basso livello, ad esempio tipi di base o raccolte.</span><span class="sxs-lookup"><span data-stu-id="4f58e-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="4f58e-151">Ad esempio, <xref:System.Collections.Generic.List%601> non genererebbe tali eventi quando un nuovo elemento viene aggiunto all'elenco e la proprietà `Count` viene modificata.</span><span class="sxs-lookup"><span data-stu-id="4f58e-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="4f58e-152">✔️ valutare la possibilità di generare eventi di notifica delle modifiche quando il valore di una proprietà viene modificato tramite forze esterne.</span><span class="sxs-lookup"><span data-stu-id="4f58e-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="4f58e-153">Se il valore di una proprietà viene modificato tramite una forza esterna (in un modo diverso dalla chiamata ai metodi sull'oggetto), la generazione di eventi indica allo sviluppatore che il valore è in corso di modifica ed è stato modificato.</span><span class="sxs-lookup"><span data-stu-id="4f58e-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="4f58e-154">Un esempio valido è la proprietà `Text` di un controllo casella di testo.</span><span class="sxs-lookup"><span data-stu-id="4f58e-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="4f58e-155">Quando l'utente digita il testo in un `TextBox`, il valore della proprietà cambia automaticamente.</span><span class="sxs-lookup"><span data-stu-id="4f58e-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
