# <a name="hashtable-and-dictionary-collection-types"></a><span data-ttu-id="2cc93-102">Tipi di raccolte Hashtable e Dictionary</span><span class="sxs-lookup"><span data-stu-id="2cc93-102">Hashtable and Dictionary Collection Types</span></span>
<span data-ttu-id="2cc93-103">La classe <xref:System.Collections.Hashtable?displayProperty=nameWithType> e le classi generiche <xref:System.Collections.Generic.Dictionary%602?displayProperty=nameWithType> e <xref:System.Collections.Concurrent.ConcurrentDictionary%602?displayProperty=nameWithType> implementano l'interfaccia <xref:System.Collections.IDictionary?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="2cc93-103">The <xref:System.Collections.Hashtable?displayProperty=nameWithType> class, and the <xref:System.Collections.Generic.Dictionary%602?displayProperty=nameWithType> and <xref:System.Collections.Concurrent.ConcurrentDictionary%602?displayProperty=nameWithType> generic classes, implement the <xref:System.Collections.IDictionary?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="2cc93-104">La classe generica <xref:System.Collections.Generic.Dictionary%602> implementa inoltre l'interfaccia generica <xref:System.Collections.Generic.IDictionary%602>.</span><span class="sxs-lookup"><span data-stu-id="2cc93-104">The <xref:System.Collections.Generic.Dictionary%602> generic class also implements the <xref:System.Collections.Generic.IDictionary%602> generic interface.</span></span> <span data-ttu-id="2cc93-105">Ogni elemento di queste raccolte è pertanto una coppia chiave-valore.</span><span class="sxs-lookup"><span data-stu-id="2cc93-105">Therefore, each element in these collections is a key-and-value pair.</span></span>  
  
 <span data-ttu-id="2cc93-106">Un oggetto <xref:System.Collections.Hashtable> è costituito da bucket che contengono gli elementi della raccolta.</span><span class="sxs-lookup"><span data-stu-id="2cc93-106">A <xref:System.Collections.Hashtable> object consists of buckets that contain the elements of the collection.</span></span> <span data-ttu-id="2cc93-107">Un bucket è un sottogruppo virtuale di elementi all'interno di <xref:System.Collections.Hashtable>, che rende più semplici e veloci le operazioni di ricerca e recupero rispetto alla maggior parte delle raccolte.</span><span class="sxs-lookup"><span data-stu-id="2cc93-107">A bucket is a virtual subgroup of elements within the <xref:System.Collections.Hashtable>, which makes searching and retrieving easier and faster than in most collections.</span></span> <span data-ttu-id="2cc93-108">Ogni bucket è associato a un codice hash, generato usando una funzione hash ed basato sulla chiave dell'elemento.</span><span class="sxs-lookup"><span data-stu-id="2cc93-108">Each bucket is associated with a hash code, which is generated using a hash function and is based on the key of the element.</span></span>  
  
 <span data-ttu-id="2cc93-109">La classe generica <xref:System.Collections.Generic.HashSet%601> è una raccolta non ordinata destinata a contenere elementi univoci.</span><span class="sxs-lookup"><span data-stu-id="2cc93-109">The generic <xref:System.Collections.Generic.HashSet%601> class is an unordered collection for containing unique elements.</span></span>  
  
 <span data-ttu-id="2cc93-110">Una funzione hash è un algoritmo che restituisce un codice hash numerico basato su una chiave.</span><span class="sxs-lookup"><span data-stu-id="2cc93-110">A hash function is an algorithm that returns a numeric hash code based on a key.</span></span> <span data-ttu-id="2cc93-111">La chiave è il valore di una proprietà dell'oggetto che viene archiviato.</span><span class="sxs-lookup"><span data-stu-id="2cc93-111">The key is the value of some property of the object being stored.</span></span> <span data-ttu-id="2cc93-112">Una funzione hash deve sempre restituire lo stesso codice hash per la stessa chiave.</span><span class="sxs-lookup"><span data-stu-id="2cc93-112">A hash function must always return the same hash code for the same key.</span></span> <span data-ttu-id="2cc93-113">È possibile per una funzione hash generare lo stesso codice hash per due chiavi diverse, ma una funzione hash che genera un codice hash univoco per ogni chiave univoca offre prestazioni migliori durante il recupero di elementi dalla tabella hash.</span><span class="sxs-lookup"><span data-stu-id="2cc93-113">It is possible for a hash function to generate the same hash code for two different keys, but a hash function that generates a unique hash code for each unique key results in better performance when retrieving elements from the hash table.</span></span>  
  
 <span data-ttu-id="2cc93-114">Ogni oggetto usato come elemento in un oggetto <xref:System.Collections.Hashtable> deve essere in grado di generare un codice hash per se stesso tramite un'implementazione del metodo <xref:System.Object.GetHashCode%2A>.</span><span class="sxs-lookup"><span data-stu-id="2cc93-114">Each object that is used as an element in a <xref:System.Collections.Hashtable> must be able to generate a hash code for itself by using an implementation of the <xref:System.Object.GetHashCode%2A> method.</span></span> <span data-ttu-id="2cc93-115">È tuttavia anche possibile specificare una funzione hash per tutti gli elementi in un oggetto <xref:System.Collections.Hashtable> usando un costruttore <xref:System.Collections.Hashtable> che accetta un'implementazione di <xref:System.Collections.IHashCodeProvider> come uno dei parametri.</span><span class="sxs-lookup"><span data-stu-id="2cc93-115">However, you can also specify a hash function for all elements in a <xref:System.Collections.Hashtable> by using a <xref:System.Collections.Hashtable> constructor that accepts an <xref:System.Collections.IHashCodeProvider> implementation as one of its parameters.</span></span>  
  
 <span data-ttu-id="2cc93-116">Quando un oggetto viene aggiunto a <xref:System.Collections.Hashtable>, viene archiviato nel bucket associato al codice hash corrispondente al codice hash dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="2cc93-116">When an object is added to a <xref:System.Collections.Hashtable>, it is stored in the bucket that is associated with the hash code that matches the object's hash code.</span></span> <span data-ttu-id="2cc93-117">Quando viene cercato un valore in <xref:System.Collections.Hashtable>, viene generato il codice hash per tale valore e viene eseguita la ricerca nel bucket associato a tale codice hash.</span><span class="sxs-lookup"><span data-stu-id="2cc93-117">When a value is being searched for in the <xref:System.Collections.Hashtable>, the hash code is generated for that value, and the bucket associated with that hash code is searched.</span></span>  
  
 <span data-ttu-id="2cc93-118">Ad esempio, una funzione hash per una stringa potrebbe prendere i codici ASCII di ogni carattere della stringa e combinarli per generare un codice hash.</span><span class="sxs-lookup"><span data-stu-id="2cc93-118">For example, a hash function for a string might take the ASCII codes of each character in the string and add them together to generate a hash code.</span></span> <span data-ttu-id="2cc93-119">La stringa "picnic" avrebbe un codice hash diverso da quello della stringa "basket", pertanto le due stringhe si troverebbero in bucket diversi.</span><span class="sxs-lookup"><span data-stu-id="2cc93-119">The string "picnic" would have a hash code that is different from the hash code for the string "basket"; therefore, the strings "picnic" and "basket" would be in different buckets.</span></span> <span data-ttu-id="2cc93-120">"Stressed" e "desserts" avrebbero invece lo stesso codice hash e si troverebbero nello stesso bucket.</span><span class="sxs-lookup"><span data-stu-id="2cc93-120">In contrast, "stressed" and "desserts" would have the same hash code and would be in the same bucket.</span></span>  
  
 <span data-ttu-id="2cc93-121">Le classi <xref:System.Collections.Generic.Dictionary%602> e <xref:System.Collections.Concurrent.ConcurrentDictionary%602> hanno la stessa funzionalità della classe <xref:System.Collections.Hashtable>.</span><span class="sxs-lookup"><span data-stu-id="2cc93-121">The <xref:System.Collections.Generic.Dictionary%602> and <xref:System.Collections.Concurrent.ConcurrentDictionary%602> classes have the same functionality as the <xref:System.Collections.Hashtable> class.</span></span> <span data-ttu-id="2cc93-122">Un oggetto <xref:System.Collections.Generic.Dictionary%602> di un tipo specifico (diverso da <xref:System.Object>) offre prestazioni migliori rispetto a un oggetto <xref:System.Collections.Hashtable> per i tipi di valore,</span><span class="sxs-lookup"><span data-stu-id="2cc93-122">A <xref:System.Collections.Generic.Dictionary%602> of a specific type (other than <xref:System.Object>) provides better performance than a <xref:System.Collections.Hashtable> for value types.</span></span> <span data-ttu-id="2cc93-123">in quanto gli elementi di <xref:System.Collections.Hashtable> sono di tipo <xref:System.Object>. Le conversioni boxing e unboxing vengono in genere eseguite quando si archivia o si recupera un tipo di valore.</span><span class="sxs-lookup"><span data-stu-id="2cc93-123">This is because the elements of <xref:System.Collections.Hashtable> are of type <xref:System.Object>; therefore, boxing and unboxing typically occur when you store or retrieve a value type.</span></span> <span data-ttu-id="2cc93-124">La classe <xref:System.Collections.Concurrent.ConcurrentDictionary%602> deve essere usata quando più thread potrebbero accedere contemporaneamente alla raccolta.</span><span class="sxs-lookup"><span data-stu-id="2cc93-124">The <xref:System.Collections.Concurrent.ConcurrentDictionary%602> class should be used when multiple threads might be accessing the collection simultaneously.</span></span>
