## <a name="remarks"></a><span data-ttu-id="b84e8-115">Note</span><span class="sxs-lookup"><span data-stu-id="b84e8-115">Remarks</span></span>  

<span data-ttu-id="b84e8-116">L'entità di codice a cui viene fatto riferimento deve essere uno dei seguenti:</span><span class="sxs-lookup"><span data-stu-id="b84e8-116">The code entity that is referenced must be one of the following:</span></span>  
  
-   <span data-ttu-id="b84e8-117">Una costante</span><span class="sxs-lookup"><span data-stu-id="b84e8-117">A constant</span></span>  
-   <span data-ttu-id="b84e8-118">Una proprietà statica</span><span class="sxs-lookup"><span data-stu-id="b84e8-118">A static property</span></span>  
-   <span data-ttu-id="b84e8-119">Un campo</span><span class="sxs-lookup"><span data-stu-id="b84e8-119">A field</span></span>

<span data-ttu-id="b84e8-121">Specifica di qualsiasi altra entità di codice, ad esempio una proprietà non statica, provoca un errore in fase di compilazione se il XAML è compilazione dal markup, o un'eccezione di analisi in fase di caricamento XAML.</span><span class="sxs-lookup"><span data-stu-id="b84e8-121">Specifying any other code entity, such as a nonstatic property, causes a compile-time error if the XAML is markup compiled, or a XAML load-time parse exception.</span></span>  

<span data-ttu-id="b84e8-122">È possibile rendere `x:Static` riferimenti a campi statici o proprietà che non si trovano nello spazio dei nomi XAML predefinito per il documento XAML corrente; tuttavia, questo richiede un mapping del prefisso.</span><span class="sxs-lookup"><span data-stu-id="b84e8-122">You can make `x:Static` references to static fields or properties that are not in the default XAML namespace for the current XAML document; however, this requires a prefix mapping.</span></span> <span data-ttu-id="b84e8-123">Gli spazi dei nomi XAML sono quasi sempre definiti per l'elemento radice del documento XAML.</span><span class="sxs-lookup"><span data-stu-id="b84e8-123">XAML namespaces are almost always defined on the root element of the XAML document.</span></span>  

<span data-ttu-id="b84e8-124">Le operazioni di ricerca per le proprietà statiche possono essere eseguite dai servizi XAML di .NET Framework e relativi reader XAML e writer XAML, quando vengono eseguite con il contesto dello schema XAML predefinito.</span><span class="sxs-lookup"><span data-stu-id="b84e8-124">The lookup operations for static properties can be performed by .NET Framework XAML Services and its XAML readers and XAML writers, when they are running with the default XAML schema context.</span></span> <span data-ttu-id="b84e8-125">In questo contesto dello schema XAML è possibile usare la reflection CLR per fornire i valori statici necessari per la costruzione dell'oggetto grafico.</span><span class="sxs-lookup"><span data-stu-id="b84e8-125">This XAML schema context can use CLR reflection to provide the necessary static values for object graph construction.</span></span> <span data-ttu-id="b84e8-126">Il `typeName` specificare è effettivamente un nome di tipo XAML, non un nome di tipo CLR, anche se si tratta essenzialmente lo stesso nome quando si usa il contesto dello schema XAML predefinito o quando si usa tutti i framework basati su CLR che implementa XAML esistenti.</span><span class="sxs-lookup"><span data-stu-id="b84e8-126">The `typeName` you specify is actually a XAML type name, not a CLR type name, although these are essentially the same name when using the default XAML schema context or when using all existing CLR-based XAML-implementing frameworks.</span></span>
