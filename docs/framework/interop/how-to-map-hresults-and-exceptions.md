# <a name="how-to-map-hresults-and-exceptions"></a><span data-ttu-id="48422-102">Procedura: eseguire il mapping di HRESULT ed eccezioni</span><span class="sxs-lookup"><span data-stu-id="48422-102">How to: Map HRESULTs and Exceptions</span></span>
<span data-ttu-id="48422-103">I metodi COM segnalano gli errori restituendo HRESULT, mentre i metodi .NET li segnalano generando eccezioni.</span><span class="sxs-lookup"><span data-stu-id="48422-103">COM methods report errors by returning HRESULTs; .NET methods report them by throwing exceptions.</span></span> <span data-ttu-id="48422-104">Il runtime gestisce la transizione tra questi due approcci.</span><span class="sxs-lookup"><span data-stu-id="48422-104">The runtime handles the transition between the two.</span></span> <span data-ttu-id="48422-105">Ogni classe di eccezione in .NET Framework viene mappata a un HRESULT.</span><span class="sxs-lookup"><span data-stu-id="48422-105">Each exception class in the .NET Framework maps to an HRESULT.</span></span>  
  
 <span data-ttu-id="48422-106">Le classi di eccezioni definite dall'utente possono specificare un HRESULT appropriato.</span><span class="sxs-lookup"><span data-stu-id="48422-106">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="48422-107">Queste classi di eccezione possono modificare dinamicamente il valore HRESULT da restituire quando viene generata l'eccezione impostando il campo **HResult** per l'oggetto eccezione.</span><span class="sxs-lookup"><span data-stu-id="48422-107">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="48422-108">Informazioni aggiuntive sull'eccezione vengono fornite al client tramite l'interfaccia **IErrorInfo**, implementata per l'oggetto .NET nel processo non gestito.</span><span class="sxs-lookup"><span data-stu-id="48422-108">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>  
  
 <span data-ttu-id="48422-109">Se si crea una classe che estende **System. Exception**, è necessario impostare il campo HRESULT durante la costruzione.</span><span class="sxs-lookup"><span data-stu-id="48422-109">If you create a class that extends **System.Exception**, you must set the HRESULT field during construction.</span></span> <span data-ttu-id="48422-110">In caso contrario, la classe di base assegna il valore HRESULT.</span><span class="sxs-lookup"><span data-stu-id="48422-110">Otherwise, the base class assigns the HRESULT value.</span></span> <span data-ttu-id="48422-111">È possibile mappare nuove classi di eccezione a un HRESULT esistente specificando il valore nel costruttore dell'eccezione.</span><span class="sxs-lookup"><span data-stu-id="48422-111">You can map new exception classes to an existing HRESULT by supplying the value in the exception's constructor.</span></span>  
  
 <span data-ttu-id="48422-112">Si noti che il runtime a volte ignorerà un `HRESULT` nei casi in cui nel thread è presente `IErrorInfo`.</span><span class="sxs-lookup"><span data-stu-id="48422-112">Note that the runtime will sometimes ignore an `HRESULT` in cases where there is an `IErrorInfo` present on the thread.</span></span>  <span data-ttu-id="48422-113">Questo comportamento può verificarsi nei casi in cui `HRESULT` e `IErrorInfo` non rappresentano lo stesso errore.</span><span class="sxs-lookup"><span data-stu-id="48422-113">This behavior can occur in cases where the `HRESULT` and the `IErrorInfo` do not represent the same error.</span></span>  
  
### <a name="to-create-a-new-exception-class-and-map-it-to-an-hresult"></a><span data-ttu-id="48422-114">Per creare una nuova classe di eccezione ed eseguirne il mapping a un HRESULT</span><span class="sxs-lookup"><span data-stu-id="48422-114">To create a new exception class and map it to an HRESULT</span></span>  
  
1.  <span data-ttu-id="48422-115">Usare il codice seguente per creare una nuova classe di eccezione denominata `NoAccessException` ed eseguirne il mapping al valore HRESULT `E_ACCESSDENIED`.</span><span class="sxs-lookup"><span data-stu-id="48422-115">Use the following code to create a new exception class called `NoAccessException` and map it to the HRESULT `E_ACCESSDENIED`.</span></span>  
  
    ```cpp  
    Class NoAccessException : public ApplicationException  
    {  
        NoAccessException () {  
        HResult = E_ACCESSDENIED;   
    }  
    }  
    CMyClass::MethodThatThrows  
    {  
    throw new NoAccessException();  
    }  
    ```  
  
 <span data-ttu-id="48422-116">Possono esistere programmi, scritti con qualsiasi linguaggio, che usano codice gestito e non gestito contemporaneamente.</span><span class="sxs-lookup"><span data-stu-id="48422-116">You might encounter a program (in any programming language) that uses both managed and unmanaged code at the same time.</span></span> <span data-ttu-id="48422-117">Ad esempio, il gestore di marshalling personalizzato nell'esempio di codice seguente usa il metodo **Marshal.ThrowExceptionForHR (int HResult)** per generare un'eccezione con un valore HRESULT specifico.</span><span class="sxs-lookup"><span data-stu-id="48422-117">For example, the custom marshaler in the following code example uses the **Marshal.ThrowExceptionForHR(int HResult)** method to throw an exception with a specific HRESULT value.</span></span> <span data-ttu-id="48422-118">Il metodo cerca il valore HRESULT e genera il tipo di eccezione appropriato.</span><span class="sxs-lookup"><span data-stu-id="48422-118">The method looks up the HRESULT and generates the appropriate exception type.</span></span> <span data-ttu-id="48422-119">Ad esempio, il valore HRESULT nel frammento di codice seguente genera **ArgumentException**.</span><span class="sxs-lookup"><span data-stu-id="48422-119">For example, the HRESULT in the following code fragment generates **ArgumentException**.</span></span>
