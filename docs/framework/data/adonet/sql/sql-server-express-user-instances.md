 <span data-ttu-id="45df3-166">Per impostazione predefinita, a tutti i membri del gruppo Windows Builtin\\Users sono concesse le autorizzazioni per connettersi all'istanza locale, nonché le autorizzazioni di lettura ed esecuzione sui binari di SQL Server.</span><span class="sxs-lookup"><span data-stu-id="45df3-166">By default all members of the Windows Builtin\Users group are granted permissions to connect on the local instance as well as read and execute permissions on the SQL Server binaries.</span></span> <span data-ttu-id="45df3-167">Dopo la verifica delle credenziali dell'utente chiamante che ospita l'istanza utente, tale utente assume il ruolo di `sysadmin` sull'istanza.</span><span class="sxs-lookup"><span data-stu-id="45df3-167">Once the credentials of the calling user hosting the user instance have been verified, that user becomes the `sysadmin` on that instance.</span></span> <span data-ttu-id="45df3-168">Per le istanze utente è abilitata solo la memoria condivisa, il che significa che sono consentite soltanto operazioni nel computer locale.</span><span class="sxs-lookup"><span data-stu-id="45df3-168">Only shared memory is enabled for user instances, which means that only operations on the local machine are possible.</span></span>  
  
 <span data-ttu-id="45df3-169">Agli utenti è necessario concedere autorizzazioni di lettura e scrittura sui file mdf e ldf specificati nella stringa di connessione.</span><span class="sxs-lookup"><span data-stu-id="45df3-169">Users must be granted both read and write permissions on the .mdf and .ldf files specified in the connection string.</span></span>

 <span data-ttu-id="45df3-173">Per evitare il danneggiamento dei dati, i database nell'istanza utente vengono aperti con accesso esclusivo.</span><span class="sxs-lookup"><span data-stu-id="45df3-173">To avoid data corruption, a database in the user instance is opened with exclusive access.</span></span> <span data-ttu-id="45df3-174">Se due istanze utente diverse condividono lo stesso database nello stesso computer, l'utente nella prima istanza deve chiudere il database prima che possa essere aperto nella seconda istanza.</span><span class="sxs-lookup"><span data-stu-id="45df3-174">If two different user instances share the same database on the same computer, the user on the first instance must close the database before it can be opened in a second instance.</span></span>
