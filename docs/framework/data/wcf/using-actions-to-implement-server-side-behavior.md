 <span data-ttu-id="76adf-164">Le azioni vengono richiamate mediante una richiesta HTTP POST.</span><span class="sxs-lookup"><span data-stu-id="76adf-164">Actions are invoked using an HTTP POST request.</span></span> <span data-ttu-id="76adf-165">L'URL specifica la risorsa seguita dal nome dell'azione.</span><span class="sxs-lookup"><span data-stu-id="76adf-165">The URL specifies the resource followed by the action name.</span></span> <span data-ttu-id="76adf-166">I parametri vengono passati nel corpo della richiesta.</span><span class="sxs-lookup"><span data-stu-id="76adf-166">Parameters are passed in the body of the request.</span></span> <span data-ttu-id="76adf-167">Se ad esempio è presente un servizio denominato MovieService che espone un'azione denominata Rate,</span><span class="sxs-lookup"><span data-stu-id="76adf-167">For example if there was a service called MovieService which exposed an action called Rate.</span></span> <span data-ttu-id="76adf-168">è possibile usare l'URL seguente per richiamare l'azione Rate in un filmato specifico:</span><span class="sxs-lookup"><span data-stu-id="76adf-168">You could use the following URL to invoke the Rate action on a specific movie:</span></span>  
  
 `http://MovieServer/MovieService.svc/Movies(1)/Rate`
  
 <span data-ttu-id="76adf-169">Movies(1) specifica il filmato che si desidera valutare e Rate specifica l'azione di valutazione.</span><span class="sxs-lookup"><span data-stu-id="76adf-169">Movies(1) specifies the movie you wish to rate and Rate specifies the Rate action.</span></span> <span data-ttu-id="76adf-170">Il valore effettivo della valutazione sarà presente nel corpo della richiesta HTTP come mostrato nell'esempio seguente:</span><span class="sxs-lookup"><span data-stu-id="76adf-170">The actual value of the rating will be in the body of the HTTP request as shown in the following example:</span></span>

```  
POST http://MovieServer/MoviesService.svc/Movies(1)/Rate HTTP/1.1   
Content-Type: application/json   
Content-Length: 20   
Host: localhost:15238  
{   
   "rating": 4   
}  
```  
  
> [!WARNING]
> <span data-ttu-id="76adf-171">Il codice di esempio precedente funzionerà solo con WCF Data Services 5.2 e versioni successive che supportano la versione light di JSON.</span><span class="sxs-lookup"><span data-stu-id="76adf-171">The above sample code will work only with WCF Data Services 5.2 and later which has support for JSON light.</span></span> <span data-ttu-id="76adf-172">Se si usa una versione precedente di WCF Data Services, è necessario specificare il tipo di contenuto json verbose come segue: `application/json;odata=verbose`.</span><span class="sxs-lookup"><span data-stu-id="76adf-172">If using an earlier version of WCF Data Services, you must specify the json verbose content-type as follows: `application/json;odata=verbose`.</span></span>

 <span data-ttu-id="76adf-174">Nel frammento di codice precedente la classe `MoviesModel` è stata generata mediante la procedura per aggiungere il riferimento a un servizio WCF Data Services in Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="76adf-174">In the code snippet above, the `MoviesModel` class was generated by using Visual Studio to Add Service Reference to a WCF Data Service.</span></span>
