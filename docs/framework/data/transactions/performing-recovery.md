 <span data-ttu-id="9c450-114">Il processo di ripristino è costituito dai due passaggi seguenti:</span><span class="sxs-lookup"><span data-stu-id="9c450-114">The recovery process consists of the following two steps:</span></span>  
  
### <a name="step-1---reenlist"></a><span data-ttu-id="9c450-115">Passaggio 1: reintegrazione</span><span class="sxs-lookup"><span data-stu-id="9c450-115">Step 1 - ReEnlist</span></span>

 <span data-ttu-id="9c450-118">Per ogni integrazione di questo tipo, il gestore di risorse richiama il metodo <xref:System.Transactions.TransactionManager.Reenlist%2A> sulla gestione transazioni.</span><span class="sxs-lookup"><span data-stu-id="9c450-118">For each such enlistment it examines, it invokes <xref:System.Transactions.TransactionManager.Reenlist%2A> on the transaction manager.</span></span> <span data-ttu-id="9c450-119">Questo metodo passa una matrice di byte contenente l'identificatore univoco <xref:System.Guid> del gestore di risorse nonché le informazioni relative all'integrazione.</span><span class="sxs-lookup"><span data-stu-id="9c450-119">This method passes on a unique <xref:System.Guid> that identifies the resource manager, as well as the enlistment's information in a byte array.</span></span> <span data-ttu-id="9c450-120">Viene quindi restituito un nuovo oggetto <xref:System.Transactions.Enlistment>.</span><span class="sxs-lookup"><span data-stu-id="9c450-120">A new <xref:System.Transactions.Enlistment> object is returned.</span></span> <span data-ttu-id="9c450-121">Se la reintegrazione non riesce e viene generata un'eccezione, il gestore di risorse deve provare in seguito a eseguire nuovamente questo passaggio.</span><span class="sxs-lookup"><span data-stu-id="9c450-121">If the reenlistment fails with an exception, the resource manager will need to retry at a later time.</span></span>  
  
 <span data-ttu-id="9c450-122">Il metodo <xref:System.Transactions.TransactionManager.Reenlist%2A> deve essere chiamato soltanto quando un gestore delle risorse viene riavviato dopo un errore.</span><span class="sxs-lookup"><span data-stu-id="9c450-122">You should only call the <xref:System.Transactions.TransactionManager.Reenlist%2A> method when a resource manager restarts from failure.</span></span> <span data-ttu-id="9c450-123">Inoltre, la reintegrazione deve riguardare esclusivamente le transazioni non ripristinate registrate da un gestore di risorse durante la fase iniziale di preparazione di un commit a due fasi.</span><span class="sxs-lookup"><span data-stu-id="9c450-123">In addition, you should only reenlist unresolved transactions logged by a resource manager during the initial Prepare phase of a two-phase commit.</span></span> <span data-ttu-id="9c450-124">Qualsiasi tentativo di chiamata a questo metodo in casi non validi può produrre risultati imprevisti.</span><span class="sxs-lookup"><span data-stu-id="9c450-124">Any attempt to call this method at invalid times can produce erroneous results.</span></span>

### <a name="step-2---completing-the-recovery"></a><span data-ttu-id="9c450-126">Passaggio 2: completamento del ripristino</span><span class="sxs-lookup"><span data-stu-id="9c450-126">Step 2 - Completing the recovery</span></span>
