## <a name="inertia"></a><span data-ttu-id="64c6a-156">Inerzia</span><span class="sxs-lookup"><span data-stu-id="64c6a-156">Inertia</span></span>  
 <span data-ttu-id="64c6a-157">Il processore di inerzia consente alle applicazioni di estrapolare percorso, orientamento e altre proprietà di un elemento dell'interfaccia utente simulando un comportamento reale.</span><span class="sxs-lookup"><span data-stu-id="64c6a-157">The inertia processor enables applications to extrapolate location, orientation, and other properties of a UI element by simulating real-world behavior.</span></span>  
  
 <span data-ttu-id="64c6a-158">Ad esempio, quando un utente applica dei gesti rapidi a un elemento, questo continua a spostarsi per poi rallentare gradualmente fino ad arrestarsi.</span><span class="sxs-lookup"><span data-stu-id="64c6a-158">For instance, when a user flicks an element, it can continue moving, decelerate, and then slowly stop.</span></span> <span data-ttu-id="64c6a-159">Il processore di inerzia implementa questo comportamento facendo in modo che i valori 2D affini (origine, scala, traslazione e rotazione) vengano modificati in un tempo specificato con una velocità di decelerazione specificata.</span><span class="sxs-lookup"><span data-stu-id="64c6a-159">The inertia processor implements this behavior by causing the affine 2-D values (origin, scale, translation, and rotation) to change over a specified time at a specified deceleration rate.</span></span>

 <span data-ttu-id="64c6a-162">L'elaborazione dell'inerzia e quella delle modifiche vengono spesso usate insieme.</span><span class="sxs-lookup"><span data-stu-id="64c6a-162">Inertia processing and manipulation processing are often used together.</span></span> <span data-ttu-id="64c6a-163">Le interfacce sono simili e gli eventi che generano in alcuni casi sono identici.</span><span class="sxs-lookup"><span data-stu-id="64c6a-163">Their interfaces are similar, and the events that they raise are (in some cases) identical.</span></span> <span data-ttu-id="64c6a-164">In generale, l'elaborazione dell'inerzia inizia quando viene completata la modifica dell'elemento dell'interfaccia utente.</span><span class="sxs-lookup"><span data-stu-id="64c6a-164">Generally, inertia processing begins when the manipulation of the UI element is completed.</span></span> <span data-ttu-id="64c6a-165">Questa operazione viene eseguita ascoltando l'evento <xref:System.Windows.Input.Manipulations.ManipulationProcessor2D.Completed> e avviando l'elaborazione dell'inerzia da questo gestore dell'evento.</span><span class="sxs-lookup"><span data-stu-id="64c6a-165">This is accomplished by listening to the <xref:System.Windows.Input.Manipulations.ManipulationProcessor2D.Completed> event and starting the inertia processing from that event handler.</span></span>
