# <a name="virtual-mode-in-the-datarepeater-control-visual-studio"></a><span data-ttu-id="4f8e9-102">Modalità virtuale nel controllo DataRepeater (Visual Studio)</span><span class="sxs-lookup"><span data-stu-id="4f8e9-102">Virtual Mode in the DataRepeater Control (Visual Studio)</span></span>
<span data-ttu-id="4f8e9-103">Quando si desidera visualizzare grandi quantità di dati tabulari in un <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> (controllo), è possibile migliorare le prestazioni impostando il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.VirtualMode%2A> proprietà `True` e gestendo in modo esplicito l'interazione del controllo con l'origine dati.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-103">When you want to display large quantities of tabular data in a <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> control, you can improve performance by setting the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.VirtualMode%2A> property to `True` and explicitly managing the control's interaction with its data source.</span></span> <span data-ttu-id="4f8e9-104">Il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> controllo fornisce diversi eventi che è possibile gestire per interagire con l'origine dati e visualizzare i dati in base alle necessità in fase di esecuzione.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-104">The <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> control provides several events that you can handle to interact with your data source and display the data as needed at run time.</span></span>  
  
## <a name="how-virtual-mode-works"></a><span data-ttu-id="4f8e9-105">Funzionamento della modalità virtuale</span><span class="sxs-lookup"><span data-stu-id="4f8e9-105">How Virtual Mode Works</span></span>  
 <span data-ttu-id="4f8e9-106">Lo scenario più comune per il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> controllo consiste nell'associare i controlli figlio del <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemTemplate%2A> a un tipo di dati di origine in fase di progettazione e consentire il <xref:System.Windows.Forms.BindingSource> per passare i dati e viceversa in base alle esigenze.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-106">The most common scenario for the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> control is to bind the child controls of the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemTemplate%2A> to a data source at design time and allow the <xref:System.Windows.Forms.BindingSource> to pass data back and forth as needed.</span></span> <span data-ttu-id="4f8e9-107">Quando si utilizza la modalità virtuale, i controlli non associati a un'origine dati e i dati vengono passati avanti e indietro all'origine dati sottostante in fase di esecuzione.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-107">When you use virtual mode, the controls are not bound to a data source, and data is passed back and forth to the underlying data source at run time.</span></span>  
  
 <span data-ttu-id="4f8e9-108">Quando il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.VirtualMode%2A> è impostata su `True`, creare l'interfaccia utente aggiungendo i controlli dal **della casella degli strumenti** anziché aggiungere controlli con associazione dal **origini dati** finestra.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-108">When the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.VirtualMode%2A> property is set to `True`, you create the user interface by adding controls from the **Toolbox** instead of adding bound controls from the **Data Sources** window.</span></span>  
  
 <span data-ttu-id="4f8e9-109">Gli eventi vengono generati in modo da controlli e, è necessario aggiungere il codice per gestire la visualizzazione dei dati.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-109">Events are raised on a control-by-control basis, and you must add code to handle the display of data.</span></span> <span data-ttu-id="4f8e9-110">Quando un nuovo <xref:Microsoft.VisualBasic.PowerPacks.DataRepeaterItem> lo scorrimento nella visualizzazione, il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValueNeeded> evento viene generato una volta per ogni controllo ed è necessario fornire i valori per ogni controllo di <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValueNeeded> gestore dell'evento.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-110">When a new <xref:Microsoft.VisualBasic.PowerPacks.DataRepeaterItem> is scrolled into view, the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValueNeeded> event is raised one time for each control and you must supply the values for each control in the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValueNeeded> event handler.</span></span>  
  
 <span data-ttu-id="4f8e9-111">Se uno dei controlli dati verrà modificato dall'utente, il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValuePushed> viene generato l'evento ed è necessario convalidare i dati e salvarla all'origine dati.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-111">If data in one of the controls is changed by the user, the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemValuePushed> event is raised and you must validate the data and save it to your data source.</span></span>  
  
 <span data-ttu-id="4f8e9-112">Se l'utente aggiunge un nuovo elemento, il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.NewItemNeeded> viene generato l'evento.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-112">If the user adds a new item, the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.NewItemNeeded> event is raised.</span></span> <span data-ttu-id="4f8e9-113">Utilizzare questo gestore di evento per creare un nuovo record nell'origine dati.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-113">Use this event's handler to create a new record in your data source.</span></span> <span data-ttu-id="4f8e9-114">Per evitare modifiche accidentali, è necessario monitorare il <xref:System.Windows.Forms.Control.KeyDown> evento per ogni controllo e chiamare <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.CancelEdit%2A> se l'utente preme il tasto ESC.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-114">To prevent unintended changes, you must also monitor the <xref:System.Windows.Forms.Control.KeyDown> event for each control and call <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.CancelEdit%2A> if the user presses the ESC key.</span></span>  
  
 <span data-ttu-id="4f8e9-115">Se l'origine dati le modifiche, è possibile aggiornare il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> controllo chiamando la <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.BeginResetItemTemplate%2A> e <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.EndResetItemTemplate%2A> metodi.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-115">If your data source changes, you can refresh the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater> control by calling the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.BeginResetItemTemplate%2A> and <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.EndResetItemTemplate%2A> methods.</span></span> <span data-ttu-id="4f8e9-116">Entrambi i metodi devono essere chiamati in ordine.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-116">Both methods must be called in order.</span></span>  
  
 <span data-ttu-id="4f8e9-117">Infine, è necessario implementare gestori eventi per il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemsRemoved> evento che si verifica quando un elemento viene eliminato e, facoltativamente, per il <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.UserDeletingItems> e <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.UserDeletedItems> che si verificano ogni volta che un utente elimina un elemento premendo il tasto CANC.</span><span class="sxs-lookup"><span data-stu-id="4f8e9-117">Finally, you must implement event handlers for the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.ItemsRemoved> event, which occurs when an item is deleted, and optionally for the <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.UserDeletingItems> and <xref:Microsoft.VisualBasic.PowerPacks.DataRepeater.UserDeletedItems> events, which occur whenever a user deletes an item by pressing the DELETE key.</span></span>
