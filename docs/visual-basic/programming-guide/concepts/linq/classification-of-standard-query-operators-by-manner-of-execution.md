<span data-ttu-id="0723d-103">Le implementazioni LINQ to Objects dei metodi degli operatori di query standard vengono eseguite in due modi principali: implementazione immediata o posticipata.</span><span class="sxs-lookup"><span data-stu-id="0723d-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="0723d-104">Gli operatori di query che usano l'esecuzione posticipata possono essere anche suddivisi in due categorie: di flusso e non di flusso.</span><span class="sxs-lookup"><span data-stu-id="0723d-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="0723d-105">Se si conosce la modalità di esecuzione dei vari operatori di query, sarà più facile capire i risultati che si ottengono da una determinata query.</span><span class="sxs-lookup"><span data-stu-id="0723d-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="0723d-106">Ciò è particolarmente vero se l'origine dei dati viene modificata oppure se si sta creando una query su un'altra query.</span><span class="sxs-lookup"><span data-stu-id="0723d-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="0723d-107">Questo argomento classifica gli operatori di query standard in base alla relativa modalità di esecuzione.</span><span class="sxs-lookup"><span data-stu-id="0723d-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="0723d-108">Modalità di esecuzione</span><span class="sxs-lookup"><span data-stu-id="0723d-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="0723d-109">Controllo immediato</span><span class="sxs-lookup"><span data-stu-id="0723d-109">Immediate</span></span>  
 <span data-ttu-id="0723d-110">L'esecuzione immediata indica che l'origine dei dati è in lettura e l'operazione viene eseguita in corrispondenza del punto del codice in cui viene dichiarata la query.</span><span class="sxs-lookup"><span data-stu-id="0723d-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="0723d-111">Tutti gli operatori di query standard che restituiscono un risultato singolo non enumerabile vengono eseguiti immediatamente.</span><span class="sxs-lookup"><span data-stu-id="0723d-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="0723d-112">Rinviata</span><span class="sxs-lookup"><span data-stu-id="0723d-112">Deferred</span></span>  
 <span data-ttu-id="0723d-113">L'esecuzione posticipata indica che l'operazione non viene eseguita in corrispondenza del punto del codice in cui viene dichiarata la query.</span><span class="sxs-lookup"><span data-stu-id="0723d-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="0723d-114">L'operazione viene eseguita solo quando la variabile di query viene enumerata, ad esempio tramite un'istruzione `For Each`.</span><span class="sxs-lookup"><span data-stu-id="0723d-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="0723d-115">Ciò significa che i risultati dell'esecuzione della query dipendono dal contenuto dell'origine dei dati nel momento in cui viene eseguita la query e non quando viene definita.</span><span class="sxs-lookup"><span data-stu-id="0723d-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="0723d-116">Se la variabile di query viene enumerata più volte, i risultati potrebbero essere diversi ogni volta.</span><span class="sxs-lookup"><span data-stu-id="0723d-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="0723d-117">Quasi tutti gli operatori query standard con tipo restituito <xref:System.Collections.Generic.IEnumerable%601> o <xref:System.Linq.IOrderedEnumerable%601> vengono eseguiti in modo posticipato.</span><span class="sxs-lookup"><span data-stu-id="0723d-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="0723d-118">Gli operatori di query che usano l'esecuzione posticipata possono essere anche suddivisi in due categorie: di flusso e non di flusso.</span><span class="sxs-lookup"><span data-stu-id="0723d-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="0723d-119">Flusso</span><span class="sxs-lookup"><span data-stu-id="0723d-119">Streaming</span></span>  
 <span data-ttu-id="0723d-120">Gli operatori di flusso non devono leggere tutti i dati di origine prima di restituire elementi.</span><span class="sxs-lookup"><span data-stu-id="0723d-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="0723d-121">In fase di esecuzione, un operatore di flusso esegue l'operazione su ogni elemento di origine che legge e restituisce l'elemento, se appropriato.</span><span class="sxs-lookup"><span data-stu-id="0723d-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="0723d-122">Un operatore di flusso continua a leggere gli elementi fino a quando non può essere prodotto un elemento di risultato.</span><span class="sxs-lookup"><span data-stu-id="0723d-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="0723d-123">Ciò significa che potrebbe essere letto più di un elemento di origine per produrre un elemento di risultato.</span><span class="sxs-lookup"><span data-stu-id="0723d-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="0723d-124">Non di flusso</span><span class="sxs-lookup"><span data-stu-id="0723d-124">Non-Streaming</span></span>  
 <span data-ttu-id="0723d-125">Gli operatori non di flusso devono leggere tutti i dati di origine prima di produrre un elemento di risultato.</span><span class="sxs-lookup"><span data-stu-id="0723d-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="0723d-126">In questa categoria rientrano operazioni quali ordinamento o raggruppamento.</span><span class="sxs-lookup"><span data-stu-id="0723d-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="0723d-127">In fase di esecuzione, gli operatori di query non di flusso leggono tutti i dati di origine, li inseriscono in una struttura di dati, eseguono l'operazione e producono gli elementi risultanti.</span><span class="sxs-lookup"><span data-stu-id="0723d-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
